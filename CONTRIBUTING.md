# Contributing to Dragon Palace Knowledge Hub

Thank you for your interest in contributing to the Dragon Palace Knowledge Hub! This document outlines the guidelines and processes for contributing to this project.

## 📋 Code of Conduct

By participating in this project, you agree to uphold our [Code of Conduct](CODE_OF_CONDUCT.md).

## 🐛 Reporting Bugs

If you find a bug, please:
1. Check if it has already been reported in the [Issues](https://github.com/your-username/dragon-palace-knowledge-hub/issues)
2. If not, create a new issue with:
   - Clear description of the problem
   - Steps to reproduce
   - Expected vs actual behavior
   - Environment details (OS, browser, etc.)

## 💡 Feature Requests

For new features:
1. Check existing issues to avoid duplicates
2. Create an issue describing:
   - The problem you're trying to solve
   - Your proposed solution
   - Any alternatives you've considered

## 🛠️ Pull Requests

### Before You Start
- Discuss significant changes in an issue first
- Ensure your code follows the existing style

### Development Setup
```bash
git clone https://github.com/your-username/dragon-palace-knowledge-hub.git
cd dragon-palace-knowledge-hub
# Make your changes
```

### Commit Guidelines
- Use descriptive commit messages
- Reference related issues (e.g., "Fix #123: Add search filters")
- Keep commits focused on single changes

### Pull Request Process
1. Fork the repository
2. Create a feature branch (`feature/your-feature` or `fix/your-fix`)
3. Make your changes and commit them
4. Push to your fork
5. Create a pull request to the main repository

Your PR should include:
- Clear description of changes
- Screenshots if UI changes are involved
- Tests if applicable
- Updated documentation

## 📚 Documentation

Documentation improvements are highly valued! Please ensure:
- New features are documented
- Existing documentation stays up-to-date
- Examples are clear and working

## 🔧 Code Style

- Follow existing code patterns in the project
- Use meaningful variable and function names
- Comment complex logic
- Keep functions focused and small

## 🧪 Testing

While this is primarily a frontend/offline application, please:
- Test your changes in multiple browsers if possible
- Ensure offline functionality works correctly
- Verify that data persistence works as expected

## 🙏 Thank You!

Your contributions help make this project better for everyone. We appreciate your time and effort!
